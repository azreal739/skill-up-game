# 01 — Product Vision

## North Star

Make engineering learning feel like joining an elite mission-control academy.

The user should not feel like they are answering training questions. They should feel like they are protecting a platform.

## Design Pillars

### 1. Story First

Every technical concept should be wrapped in a mission context.

Bad:
> What is a TypeScript interface?

Good:
> The mission dashboard is receiving customer data from a Java API. The current model is too loose and defects are escaping into production. Define a safer interface and explain what it protects.

### 2. Real Stack, Real Problems

The content must use the team's stack:

- Angular components and templates.
- TypeScript type safety.
- Nx apps and libraries.
- SCSS layout and maintainability.
- Zod schema validation.
- Java REST contracts.
- S3 and CloudFront deployment concepts.
- Cloudflare edge/caching/security concepts.

### 3. Gradual Complexity

Start small. Increase complexity mission by mission.

The first missions should be approachable. Later missions should feel like real-world engineering judgement.

### 4. Failure Teaches

Wrong answers should not embarrass or punish. They should create consequences and teaching moments.

Examples:
- Technical debt meter increases.
- Platform stability drops.
- Incident timer advances.
- AI mentor explains the missed concept.
- A follow-up recovery challenge appears.

### 5. Beautiful Feedback

Every meaningful action should feel good.

Examples:
- XP fly-ups.
- Badges unlocking.
- Pipeline animations.
- Dashboard status changes.
- Sound effects.
- Mission briefings.
- Confetti when production is restored.

### 6. Extensible Content

The engine should support new campaigns as content packs.

Adding a new mission should primarily involve adding JSON/TypeScript data, not rewriting the engine.

## Product Tone

The game should feel:

- Encouraging.
- Smart.
- Slightly dramatic.
- Futuristic.
- Team-oriented.
- Professional enough for work.
- Fun enough that juniors ask to play again.

## World Concept

The Academy trains engineers to protect and evolve a large digital platform. The platform serves many customers and depends on Angular applications, Java services, cloud hosting, CDNs and edge networks.

The player starts as a Graduate Engineer and enters increasingly realistic simulations.

## Player Fantasy

The player should feel like:

- A new engineer learning the ropes.
- A detective investigating broken systems.
- A platform defender restoring production.
- A future technical lead making responsible trade-offs.

## Inspiration

The product may take inspiration from:

- Escape rooms.
- RPG progression.
- Mission-control dashboards.
- Incident simulations.
- Duolingo-style progression.
- Code review exercises.
- Dungeons & Dragons-style facilitation.

Do not copy any proprietary game. Use these only as conceptual inspiration.
