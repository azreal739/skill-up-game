# 16 — Accessibility and Inclusion

## Principle

The game should be fun and accessible. Learning should not depend on perfect vision, hearing, fast reflexes or comfort with animations.

## Requirements

### Keyboard Support

All controls must be keyboard accessible.

### Focus States

Visible focus indicators must be present.

### Colour Contrast

Ensure sufficient contrast for text and important UI states.

### Reduced Motion

Provide reduced motion mode.

### Audio Independence

No important information should be audio-only.

### Captions

If voice narration is added later, captions are required.

### Text Scaling

UI should tolerate larger text.

### Colour Blind Support

Do not rely on colour alone for severity or correctness.

Use icons, labels and text.

### Inclusive Feedback

Avoid language like:
- "Obviously"
- "Easy"
- "You failed"
- "Wrong again"

Use:
- "Good attempt"
- "Look at the runtime boundary"
- "This is a common production issue"
- "Try another approach"
