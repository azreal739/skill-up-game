# 19 — Definition of Done

## Product DoD

The game is done when:

- It feels like a game, not a static quiz.
- It has a working campaign.
- It uses the team's technology stack as learning content.
- It supports progression.
- It has contextual help.
- It gives meaningful feedback.
- It persists progress.
- It has polish through animation and sound.
- It is accessible.
- It is easy to extend.

## Engineering DoD

Code is done when:

- It builds.
- It passes tests.
- It is formatted.
- It follows Angular/Nx structure.
- It separates content from engine.
- It validates content with Zod.
- It has no obvious dead code.
- It has meaningful names.
- It avoids unnecessary complexity.

## Content DoD

Mission content is done when:

- Story is clear.
- Learning objective is clear.
- Challenge is realistic.
- Correct answer is explained.
- Incorrect answer teaches.
- Help links exist.
- Hints exist.
- Rewards and consequences exist.

## UX DoD

UX is done when:

- Player always knows what to do.
- Player can get help.
- Feedback is clear.
- Important state is visible.
- Motion can be reduced.
- Audio can be muted.
- Keyboard usage works.

## Final Acceptance Scenario

A new junior engineer should be able to:

1. Open the app.
2. Understand the story.
3. Create a profile.
4. Start Campaign 1.
5. Complete at least three missions.
6. Learn from feedback.
7. Use Help Centre.
8. Earn XP.
9. Close the app.
10. Return and continue progress.

If this works smoothly, the first vertical slice is successful.
